## Definition

Software engineering is the disciplined application of engineering principles to the design, development, testing, and maintenance of software systems.

## Why It Matters

Software engineering practices enable teams to build reliable, maintainable, and scalable systems while managing complexity and change over time.

## Key Principles

- Separation of concerns — divide systems into distinct sections
- DRY — avoid duplicating knowledge across the codebase
- YAGNI — do not build features until they are needed

## How It Works

1. Gather and analyse requirements from stakeholders.
2. Design the system architecture and component interfaces.
3. Implement the design iteratively with continuous feedback.
4. Test at multiple levels (unit, integration, system).
5. Deploy and monitor in production, feeding learnings back into development.

## Examples

- Agile methodologies for iterative delivery.
- Design patterns for reusable solutions to common problems.
- Code review practices for quality assurance.

## Common Pitfalls

- Big-bang releases instead of incremental delivery.
- Technical debt accumulation without planned repayment.
- Over-engineering solutions for problems that may never arise.

## Trade-offs

| Aspect | Benefit | Cost |
| --- | --- | --- |
| Agile | Fast feedback, adaptability | Requires discipline to avoid chaos |
| Waterfall | Clear milestones | Slow to respond to change |
| Microservices | Independent deployment | Distributed system complexity |
